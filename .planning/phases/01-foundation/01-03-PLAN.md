```
Extractive summarization is a technique in natural language processing that creates a summary by selecting and extracting sentences or passages directly from the source document. Unlike abstractive summarization, which generates new text, extractive methods preserve the original wording and meaning by identifying the most important spans of text.

The task of extractive summarization has been studied since the 1950s. Early systems used simple heuristics such as word frequency and sentence position to determine importance. A sentence that appears near the beginning of a document, or that contains frequently used words, was considered more likely to be summary-worthy.

Graph-based methods, introduced in the early 2000s, represented a significant advance. In these approaches, a document is modeled as a graph where nodes represent sentences and edges represent similarity between sentences. The importance of each sentence is then computed using a variant of PageRank, the algorithm originally developed to rank web pages by Google. Sentences that are similar to many other sentences in the document receive higher scores, because they are considered to capture the central themes.

LexRank, published by Erkan and Radev in 2004, is one of the most cited graph-based extractive summarization algorithms. LexRank computes sentence similarity using TF-IDF weighted cosine similarity, then applies a modified PageRank over the sentence similarity graph. The algorithm is unsupervised, requiring no training data, and produces results competitive with supervised methods on standard benchmarks.

TextRank, introduced by Mihalcea and Tarau in 2004, is another influential graph-based method. Like LexRank, it builds a graph of sentence similarities and applies a PageRank variant. TextRank uses a simpler similarity metric based on word overlap rather than TF-IDF, making it faster to compute while still producing high-quality summaries.

Both LexRank and TextRank select the top-scoring sentences and then reorder them to match their original position in the document. This document-order preservation ensures that the summary reads naturally and maintains the logical flow of the original text.

Evaluation of extractive summarization systems typically uses the ROUGE metric, which measures the overlap of n-grams between the system-generated summary and human-written reference summaries. ROUGE-1 measures unigram overlap, ROUGE-2 measures bigram overlap, and ROUGE-L measures the longest common subsequence.

Modern neural approaches such as BERT-based sentence embeddings and transformer models have pushed the state of the art, but graph-based methods remain popular for their efficiency, interpretability, and lack of dependency on large labeled training sets. For many practical applications, a well-implemented LexRank or TextRank system provides a strong, fast baseline.

Extractive summarization has applications in news article condensation, meeting transcript summarization, legal document review, and academic research assistance. The technique is particularly useful when preserving the original author's phrasing is important, such as in legal and medical contexts where paraphrasing could alter meaning.

The main limitation of extractive methods is their inability to synthesize information across sentences. If the key insight of a document requires combining information from multiple disconnected paragraphs, extractive summarization may miss it. Abstractive methods address this by generating new sentences, but at the cost of potentially introducing factual errors.
```

Today I want to talk about how we think about text summarization at scale.
When you have a million documents and you need to surface the key points, you cannot afford to call an LLM for every single document.
The cost and latency make that approach impractical at any significant volume.
So we looked at classic extractive methods, specifically LexRank and TextRank, and we were surprised by how well they perform.
These are deterministic algorithms with no GPU requirement and no API key.
You can run them on a laptop and summarize a thousand documents a minute.
The core idea behind both algorithms is the same: model the document as a graph.
Each sentence becomes a node.
If two sentences are similar, you draw an edge between them.
Then you run PageRank over that graph.
The sentences that are most similar to the most other sentences bubble up to the top.
Those are your summary sentences.
LexRank uses TF-IDF cosine similarity to measure sentence relatedness.
TextRank uses a simpler word overlap measure.
In practice, for most documents, the results are nearly identical.
Where they differ is on highly technical text with specialized vocabulary.
LexRank handles jargon better because TF-IDF naturally downweights common words and upweights rare ones.
TextRank can over-weight sentences that repeat common words.
For our use case, scientific abstracts and engineering design documents, LexRank was the clear winner.
We wrote our implementation in Go because our data pipeline is already Go.
The implementation took about three hundred lines total for both algorithms.
Unit testing was straightforward because the algorithms are deterministic.
Same input always produces the same output.
That property makes testing and debugging much easier than working with probabilistic models.
We open-sourced the implementation and it has been running in production for six months without issues.
If you need fast, free, offline text summarization, I strongly recommend looking at these graph-based methods before reaching for a large language model.

Natural language processing, commonly abbreviated as NLP, is a subfield of linguistics, computer science, and artificial intelligence concerned with the interactions between computers and human language, in particular how to program computers to process and analyze large amounts of natural language data.

The history of natural language processing generally started in the 1950s, although work can be found from earlier periods. In 1950, Alan Turing published an article titled "Computing Machinery and Intelligence" which proposed what is now called the Turing test as a criterion of intelligence. The Georgetown experiment in 1954 involved fully automatic translation of more than sixty Russian sentences into English. The authors claimed that within three to five years, machine translation would be a solved problem. However, real progress was much slower, and after the ALPAC report in 1966, which found that ten-year-long research had failed to fulfill the expectations, funding for machine translation was dramatically reduced.

Little further research in machine translation was conducted until the late 1980s when the first statistical machine translation systems were developed. Between 1988 and 1990, IBM Research developed a statistical approach to machine translation based on the noisy-channel model. This work introduced the concept of training translation models from parallel corpora, which became the dominant paradigm for the following two decades.

In the 1970s, many programmers began to write conceptual ontologies, which structured real-world information into computer-understandable data. Examples are MARGIE, SAM, PAM, TaleSpin, QUALM, Politics, and Plot Units. During this period, the emphasis shifted toward knowledge representation and reasoning. Researchers built hand-crafted rule systems that encoded linguistic knowledge explicitly. These expert systems were impressive demonstrations but were brittle and did not generalize well to new text domains.

The 1980s brought a major change in the methodological approach to natural language processing. Researchers began to move away from hand-coded rules toward machine learning methods, especially as computational power increased and large corpora became available. The Penn Treebank, created at the University of Pennsylvania, provided millions of words of parsed English text that could be used to train statistical parsers.

During the 1990s, many of the classical NLP tasks, such as part-of-speech tagging, syntactic parsing, and named entity recognition, were effectively solved using statistical approaches based on Hidden Markov Models and related techniques. The Brown corpus and the Penn Treebank made supervised learning for NLP practical at scale.

The turn of the millennium saw the rise of machine learning approaches applied to NLP at all levels of granularity. Support vector machines, logistic regression, and naive Bayes classifiers were widely used. The shared task format, pioneered by the Message Understanding Conferences and later by CoNLL, became the dominant way to advance the field by providing standard datasets and evaluation metrics.

Word embeddings, introduced in their modern form by Mikolov and colleagues in the Word2Vec paper of 2013, represented a breakthrough in distributed representations of words. Instead of treating words as discrete symbols, word embeddings map each word to a dense vector in a continuous space where semantic similarity is captured by geometric proximity. The GloVe embeddings from Stanford, published in 2014, refined this approach by incorporating global corpus statistics.

The development of sequence-to-sequence models using recurrent neural networks, introduced by Sutskever, Vinyals, and Le in 2014, opened the door to neural machine translation. The encoder-decoder architecture with attention, described by Bahdanau, Cho, and Bengio in the same year, allowed models to focus on relevant parts of the input when generating each word of the output. These models achieved state-of-the-art results on machine translation and were quickly applied to summarization, question answering, and dialogue generation.

The introduction of the Transformer architecture by Vaswani and colleagues in 2017 in the paper "Attention is All You Need" was arguably the most significant milestone in NLP since the introduction of word embeddings. Transformers replaced recurrent neural networks with self-attention mechanisms, enabling much more efficient parallel training on modern hardware. The Transformer became the foundation for all subsequent large language models.

BERT, introduced by Devlin and colleagues at Google in 2018, demonstrated that pre-training a large Transformer model on massive unlabeled text data and then fine-tuning it on specific tasks consistently produced state-of-the-art results across a wide range of NLP benchmarks. BERT and its variants, including RoBERTa, ALBERT, and DistilBERT, dominated NLP leaderboards for several years.

The GPT series of models from OpenAI, beginning with GPT-1 in 2018 and continuing through GPT-2, GPT-3, and subsequent versions, demonstrated that autoregressive language models trained on web-scale text could perform impressively on tasks they were not explicitly trained for, simply by providing task instructions in the input. This capability, known as in-context learning or few-shot learning, fundamentally changed the way practitioners approach NLP tasks.

Large language models based on the Transformer architecture are now used in production systems for machine translation, question answering, text summarization, code generation, sentiment analysis, named entity recognition, relation extraction, and many other tasks. The scale of these models, measured in billions of parameters and trained on trillions of tokens of text, has produced emergent capabilities that were not anticipated by researchers working at smaller scales.

Despite the impressive capabilities of large language models, several challenges remain. Hallucination, the tendency of language models to generate plausible-sounding but factually incorrect statements, is a significant problem for high-stakes applications. The computational cost of training and serving large models limits their accessibility. Privacy and intellectual property concerns arise when models are trained on internet-scraped data that may contain sensitive information or copyrighted material.

Classical NLP methods, including rule-based systems, statistical models, and smaller neural networks, remain relevant for many production applications where efficiency, interpretability, and determinism are priorities. Named entity recognition, part-of-speech tagging, dependency parsing, and coreference resolution are often handled by efficient statistical models that run in milliseconds on a single CPU core.

Extractive summarization, which selects and returns sentences from the original document without modification, is particularly well-suited to classical methods because the correctness of the extracted sentences is guaranteed by their origin. An extractive system cannot introduce hallucinations in the same way an abstractive system can, because it only selects text that was already present. This property makes extractive summarization attractive for legal, medical, and financial applications where the original source text must be preserved exactly.

The field of NLP is advancing rapidly, with new models, datasets, benchmarks, and techniques appearing every year. The open-source community has played a major role in democratizing access to state-of-the-art NLP tools, with libraries such as Hugging Face Transformers, spaCy, NLTK, and Stanford CoreNLP making cutting-edge research accessible to practitioners worldwide.

The intersection of NLP with other fields, including computer vision, speech recognition, robotics, and bioinformatics, continues to produce new applications and challenges. Multimodal models that process text together with images, audio, or structured data represent the current frontier of applied NLP research.

As NLP systems become more capable and more widely deployed, questions of fairness, bias, and accountability become increasingly important. Language models trained on historical text may encode and amplify existing biases related to gender, race, and other demographic categories. Researchers and practitioners are actively developing methods to measure, mitigate, and audit these biases.

The governance of large language models, including questions of access, transparency, and liability, is an active area of policy discussion. The rapid deployment of generative AI systems has prompted regulatory attention in multiple jurisdictions, leading to calls for standards around safety testing, disclosure of training data, and accountability for model outputs.

Looking ahead, the integration of language models with external knowledge sources, memory systems, and reasoning engines is seen as a path toward more reliable and grounded AI assistants. Retrieval-augmented generation, which combines neural generation with information retrieval from a curated knowledge base, is one promising approach that can reduce hallucination while maintaining the fluency and flexibility of large language models.

The future of natural language processing will likely involve a diverse ecosystem of models, from small efficient classifiers running on edge devices to massive multimodal systems running in cloud data centers. Classical methods, statistical models, and large neural networks will coexist, each suited to different constraints of latency, compute, accuracy, and interpretability. Understanding all of these approaches remains valuable for practitioners building real-world NLP systems.

package summarizer

import (
	"os"
	"path/filepath"
	"runtime"
	"testing"
)

// repoRoot returns the repository root directory using the location of this test file.
// This allows the tests to locate test-data/ regardless of working directory.
func repoRoot(t *testing.T) string {
	t.Helper()
	// This file is at internal/summarizer/integration_test.go
	// repo root is two levels up
	_, filename, _, ok := runtime.Caller(0)
	if !ok {
		t.Fatal("runtime.Caller failed")
	}
	// filename = .../internal/summarizer/integration_test.go
	// filepath.Dir twice gives repo root
	return filepath.Dir(filepath.Dir(filepath.Dir(filename)))
}

func readTestFile(t *testing.T, name string) string {
	t.Helper()
	path := filepath.Join(repoRoot(t), "test-data", name)
	data, err := os.ReadFile(path)
	if err != nil {
		t.Fatalf("could not read test fixture %s: %v", path, err)
	}
	return string(data)
}

func TestSummarize_WikipediaEn(t *testing.T) {

	}
}
