## Existing pkg/tldt Surface

From pkg/tldt/tldt.go:
- Current Fetch: `func Fetch(url string, opts FetchOptions) (string, error)` returns only text
- Current sentinel errors: `ErrSSRFBlocked`, `ErrRedirectLimit` (re-exported from fetcher)
- Existing types: SummarizeOptions, DetectOptions, FetchOptions, PipelineOptions
- Existing result types: Result, DetectResult, SanitizeReport, PipelineResult

## Context Decisions

**D-02 (Hybrid approach for Fetch):** Return `(FetchResult, error)` where FetchResult contains Text, StatusCode, ContentType, FinalURL — metadata for middleware consumers who need it, while keeping text extraction the primary use case.

**D-03 (Error wrapping):** Wrap errors with `fmt.Errorf("tldt.FunctionName: %w", err)`. Add sentinel errors `ErrHTTPError` (non-2xx status) and `ErrNonHTML` (content type not HTML) for type-checking.

**D-01 (Integration test coverage):** Test at pkg/tldt layer covering: all 4 algorithms (lexrank, textrank, graph, ensemble), empty input handling, invalid algorithm, injection detection, Unicode sanitization, Fetch with SSRF block. Import only pkg/tldt — no internal packages.
</context>
