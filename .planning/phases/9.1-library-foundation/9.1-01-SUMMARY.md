# Phase 9.1 Plan 01: Enhance pkg/tldt Library Surface

## Summary

Enhanced the public API surface in `pkg/tldt` to support complete library embedding use cases. Added `FetchResult` type with HTTP metadata for middleware consumers, defined sentinel errors (`ErrHTTPError`, `ErrNonHTML`) for type-safe error checking, and expanded integration tests to cover all edge cases and error scenarios per D-01, D-02, D-03 decisions.

## What Was Built

### 1. FetchResult Type (D-02)

Added structured result type for `Fetch()` with full HTTP metadata:

```go
type FetchResult struct {
    Text        string // Extracted article text
    StatusCode  int    // HTTP status code (after redirects)
    ContentType string // Response Content-Type header
    FinalURL    string // Final URL after all redirects
}
```

This enables external Go programs to consume tldt as a pure library dependency with access to fetch metadata for middleware use cases.

### 2. Sentinel Errors (D-03)

Added two new sentinel errors for type-safe error checking:

```go
var (
    ErrHTTPError = errors.New("tldt: HTTP error")     // Non-2xx status
    ErrNonHTML   = errors.New("tldt: not HTML content") // Wrong content-type
)
```

All errors are now wrapped with context: `fmt.Errorf("tldt.Fetch: %w", err)`. Callers can use `errors.Is(err, ErrHTTPError)` for type checking.

### 3. Comprehensive Integration Tests (D-01)

Added 9 new edge case integration tests covering:

| Test | Coverage |
|------|----------|
| TestSummarize_EmptyInput | Empty string returns empty summary, zero tokens |
| TestSummarize_WhitespaceOnly | Whitespace-only input handling |

**Key constraint honored:** All tests import **only `pkg/tldt`** — zero `internal/` package dependencies. This proves the public API contract is complete.

## API Changes

### Breaking Change: Fetch Signature

**Before:**
```go
func Fetch(url string, opts FetchOptions) (string, error)
```

**After:**
```go
func Fetch(url string, opts FetchOptions) (FetchResult, error)
```

**Migration:** Callers change `text, err := tldt.Fetch(...)` to `result, err := tldt.Fetch(...)` then access `result.Text`.
