<decisions_made>

- **Ensemble = simple average**: Both algorithms produce ~1-summing stationary distributions via power iteration. Element-wise average is valid without renormalization.
- **Two-pass Makefile for coverage**: `-coverprofile` flag interferes with `GOCOVERDIR` subprocess collection. Pass 1 = unit coverage with `-coverprofile`; Pass 2 = `GOCOVERDIR` only (no `-coverprofile`).
- **Removed dead code in rouge.go**: `sysTotal==0` guard is unreachable (non-empty `ngramCounts` implies `len(sys)>=n`). Custom `max()` removed (Go 1.26 builtin).
- **ROUGE output → stderr only**: Consistent with `--verbose` / `--explain` / token stats — stdout must remain pipe-safe for downstream tools.
- **IDF min/max test corpus**: Used "aardvark" corpus (medium frequency=2/4 sentences) so both v<IDFMin and v>IDFMax branches trigger. Prior "animals" corpus had first-alphabetical word at max IDF, blocking the high branch.
</decisions_made>

## Critical Anti-Patterns (do NOT repeat these)
- **`-coverprofile` + `GOCOVERDIR` in same `go test` invocation**: They interfere. Two-pass is required (unit pass with `-coverprofile`, subprocess pass with `GOCOVERDIR` only, no `-coverprofile`).
- **IDF branch corpus design**: First-alphabetically-sorted word determines IDFMin starting value. Design corpus so that word has medium frequency, not extreme, so both update branches (lower and higher) can trigger.
- **`trRowNormalize` uses row count not column count for uniform**: When writing matrix tests, uniform value = 1/n where n = number of rows, not columns.
